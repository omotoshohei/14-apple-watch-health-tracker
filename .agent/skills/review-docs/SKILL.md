# ドキュメントレビューの実行スキル (Antigravity版)

このスキルは、プロジェクト内の設計書や要件定義書などのドキュメントを読み込み、完全性・具体性・一貫性・測定可能性の観点から詳細なレビューを行い、構造化されたレポートを作成するためのガイドです。

## 概要

指定されたドキュメントを精読し、既存の設計や他のドキュメント（`docs/product-requirements.md` など）との整合性を含めて評価を行います。

---

## 実行手順

### ステップ 1: ドキュメントの確認と読み込み
1. 指定されたドキュメント（例: `docs/product-requirements.md`）が存在するか確認します。
2. Antigravity のファイル閲覧機能を使用して、ドキュメントの全内容を読み込みます。

### ステップ 2: 詳細レビューの実施
以下の評価基準に沿って、ドキュメントを多角的に分析・評価します。

#### 評価観点
1. **完全性 (Completeness)**:
   - 必要なセクションや項目が不足なく全て含まれているか。
2. **具体性 (Specificity)**:
   - 曖昧な表現、推測に基づく記述、未確定のプレースホルダーがないか。
   - CLIインターフェースやAPI仕様がある場合、具体的なパラメータや動作例が記載されているか。
3. **一貫性 (Consistency)**:
   - `docs/` 配下にある他の永続ドキュメント（機能設計、アーキテクチャ、開発ガイドライン等）や、プロジェクト全体の設計方針と矛盾していないか。
4. **測定可能性 (Measurability)**:
   - プロダクト要求定義書 (PRD) などの場合、成功指標 (KPI) や非機能要件のパフォーマンス基準が具体的な数値で定義されているか。

### ステップ 3: レビューレポートの出力
レビュー結果を以下のフォーマットにまとめてユーザーに提示します。

```markdown
# ドキュメントレビュー結果

## ドキュメント: [ファイル名または相対パス]

### 主な改善点

1. **[改善項目名]** (優先度: 高/中/低)
   - **現状**: [ドキュメントにおける現在の記述・課題]
   - **理由**: [なぜ改善が必要なのか]
   - **推奨される修正案**: [具体的な記述例や修正コード]

2. **[改善項目名]** (優先度: 高/中/低)
   - ...

### 総合評価

[1-5]/5
- **5 (優れている)**: そのまま実装・設計の次のフェーズに進められる極めて高い完成度。
- **4 (良好)**: 軽微な修正で完了する。
- **3 (普通)**: 一部の主要セクションに具体化や整合性の調整が必要。
- **2 (要改善)**: 多くの重要項目が不足している、または内容が曖昧。
- **1 (不適合)**: 全体的な再考が必要。

### 次のアクション

- [ ] [推奨対応1]
- [ ] [推奨対応2]
```

## 注意事項

- レビューの客観性と精度を高めるため、[development-guidelines](../development-guidelines/SKILL.md) や他のドキュメントで定義されているプロジェクト固有のルールを必ず事前確認してください。
- 改善点には、単なる問題提起にとどまらず、**具体的な修正文案や代替の記述例**を提示するようにしてください。
